2. The benefit of these type hints is the runtime cost, and the fact it makes Python way more flexible as a language, and faster to write. The only downside on python is that it doesn't enforce the types on its own. You need to run mypy in order to enforce types. So there is a good side and a bad side. But if python did enforce types like Java, python would catch way more errors automatically, and would become way more of a strict language. 
3. For the case of Lab 18 and the record with a set of named fields I would probably choose TypedDict. Since we are using a fixed structure, and a TypedDict makes sure to enforce specific keys so it's easy to look up, and it is able to catch typos which is important for a record. For the case of Lab 20 with a mapping from variable keys to values I would probably use dict. I would use dict for mapping because the keys aren't fix to a variable, and these keys are able to change as they need to. This is important for mapping, and is why dict would be better for this case.
